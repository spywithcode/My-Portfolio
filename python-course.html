a = 10

a + b → 13
a - b → 7
a * b → 30

a % b → 1

Important Logic:

  • / operator hamesha float value deta hai
  • // operator decimal part ko hata deta hai, rounding nahi karta
  • % operator remainder nikalne ke liye use hota hai

Example 1: Addition and Subtraction

x = 20
y = 5

Example 2: Multiplication

4.2 Relational (Comparison) Operators

Relational Operators ka use do values ko compare karne ke liye hota hai.

In operators ka result hamesha True ya False hota hai.

  • == → Equal to (barabar hai ya nahi)
  • != → Not equal to (barabar nahi hai)
  • > → Greater than (bada hai)
  • < → Less than (chhota hai)
  • >= → Greater than or equal to
  • <= → Less than or equal to

Basic Example:

a = 10
b = 20

a == b → False
a != b → True
a > b → False
a < b → True
a >= b → False
a <= b → True

Important Logic:

  • Relational operators sirf comparison karte hain
  • Ye condition statements (if, while) me bahut use hote hain

Example 1: Greater than Operator

x = 15
y = 10

x > y → True

Example 2: Less than Operator

a = 5

Example 3: Equal to Operator

Example 4: Not Equal Operator

Example 5: Real Life Condition Check

age = 18

age >= 18 → True

Person voting ke liye eligible hai.

Common Mistake (Very Important):

  • = → Assignment operator (value assign karta hai)
  • == → Comparison operator (value compare karta hai)

Galti se = ki jagah == aur vice-versa use karna beginners ki sabse common mistake hoti hai.

4.3 Logical Operators

Logical Operators ka use multiple conditions ko combine karne ke liye hota hai. Ye operators decide karte hain ki final condition True hogi ya False.

  • and → Dono conditions True honi chahiye
  • or → Koi ek condition True ho to result True hota hai
  • not → Condition ka result reverse karta hai

Basic Example:

a = 10
b = 5

a > 5 and b < 10 → True

Explanation:
a > 5 → True (kyunki 10, 5 se bada hai)
b < 10 → True (kyunki 5, 10 se chhota hai)
Dono conditions True hain, isliye and ka final result → True

a > 15 or b < 10 → True

Explanation:
a > 15 → False (10, 15 se bada nahi hai)
b < 10 → True (5, 10 se chhota hai)
or operator me ek bhi condition True ho to result → True

not (a > 5) → False

Explanation:
a > 5 → True
not operator True ko False bana deta hai
Isliye final result → False

Example 1: and Operator

x = 8
y = 4

x > 5 and y < 10 → True

Explanation:
x > 5 → True
y < 10 → True
Dono True hone ki wajah se result → True

Example 2: or Operator

marks = 35

marks >= 33 or marks == 32 → True

Explanation:
marks >= 33 → True (student pass hai)
marks == 32 → False
Ek condition True hai, isliye or ka result → True

Example 3: not Operator

is_raining = False

not is_raining → True

Explanation:
is_raining ki value False hai
not operator False ko True bana deta hai

Example 4: Combined Conditions

age = 20
has_id = True

age >= 18 and has_id → True

Explanation:
age >= 18 → True (eligible age)
has_id → True
Dono conditions True hain, isliye final result → True

Example 5: Login Condition

username_correct = True
password_correct = False

username_correct and password_correct → False

Explanation:
Username sahi hai → True
Password galat hai → False
and operator me dono True hone chahiye, isliye result → False
Login fail ho jayega

4.4 Assignment Operators

Assignment Operators ka use value assign karne aur usi value ko update karne ke liye hota hai.

Ye operators variable ki existing value par operation karke result ko usi variable me store kar dete hain.

  • = → Assign value
  • += → Add & assign
  • -= → Subtract & assign
  • *= → Multiply & assign
  • /= → Divide & assign
  • //= → Floor divide & assign
  • %= → Modulus & assign
  • **= → Power & assign

Basic Example:

a = 10

a += 5 → 15

Explanation:
a += 5 ka matlab hai: a = a + 5
Pehle a ki value 10 thi
10 + 5 = 15
Isliye a ki new value → 15

a *= 2 → 30

Explanation:
a *= 2 ka matlab hai: a = a * 2
Ab a ki value 15 thi
15 × 2 = 30
Isliye a ki new value → 30

Important Logic:

a += b ≡ a = a + b

Assignment operator hamesha right side ka result left side wale variable me store karta hai.

Example 1: Subtract & Assign

x = 20

x -= 5 → 15

Explanation:
x -= 5 ≡ x = x - 5
20 - 5 = 15

Example 2: Divide & Assign

a = 20

a /= 4 → 5.0

Explanation:
a /= 4 ≡ a = a / 4
Normal division hoti hai, isliye result float hota hai

Example 3: Floor Divide & Assign

b = 17

b //= 3 → 5

Explanation:
b //= 3 ≡ b = b // 3
17 / 3 = 5.66
Floor division sirf integer part rakhta hai

Example 4: Modulus & Assign

n = 10

n %= 3 → 1

Explanation:
n %= 3 ≡ n = n % 3
10 ko 3 se divide karne par remainder 1 aata hai

Example 5: Power & Assign

p = 2

p **= 4 → 16

Explanation:
p **= 4 ≡ p = p ** 4
2 ki power 4 = 16

4.5 Bitwise Operators

Bitwise Operators numbers ke binary (0 aur 1) representation par kaam karte hain. Ye operators har bit par operation perform karte hain.

  • & → Bitwise AND
  • | → Bitwise OR
  • ^ → Bitwise XOR
  • ~ → Bitwise NOT
  • << → Left Shift
  • >> → Right Shift

Basic Example:

a = 5 (Binary: 0101)
b = 3 (Binary: 0011)

a & b → 1

Explanation:
a = 0101
b = 0011
AND operation me dono bits 1 hone chahiye
Result = 0001 (Decimal: 1)

a | b → 7

Explanation:
OR operation me agar koi ek bit 1 ho to result 1 hota hai
0101
0011
----
0111 (Decimal: 7)

a ^ b → 6

Explanation:
XOR me same bits → 0, different bits → 1
0101
0011
----
0110 (Decimal: 6)

~a → -6

Explanation:
a = 5 (00000101)
~ operator har bit ko reverse karta hai
Result two’s complement form me negative hota hai
Formula: ~n = -(n + 1)
~5 = -(5 + 1) = -6

a << 1 → 10

Explanation:
Left shift me bits left ki taraf shift hoti hain
0101 << 1 → 1010 (Decimal: 10)
Har left shift ka matlab number × 2 hota hai

a >> 1 → 2

Explanation:
Right shift me bits right ki taraf shift hoti hain
0101 >> 1 → 0010 (Decimal: 2)
Har right shift ka matlab number ÷ 2 hota hai

Important Points:

  • Bitwise operators binary level par kaam karte hain
  • Bitwise operators Logical operators se alag hote hain
  • Logical operators (and, or, not) True/False par kaam karte hain
  • Bitwise operators (&, |, ^) bits par kaam karte hain

4.6 Membership Operators

In operators ka result hamesha True ya False hota hai.

  • in → Agar value sequence me present ho
  • not in → Agar value sequence me present na ho

Explanation:

Isliye result → True

5 not in nums → True

Explanation:

Example 2: Tuple me Membership

t = (10, 20, 30)

Explanation:

Example 3: Set me Membership

s = {1, 3, 5, 7}

3 in s → True

Explanation:
Set unordered hota hai, par membership check fast hota hai

Example 4: Dictionary me Membership

student = {"name": "Ram", "age": 20}

"name" in student → True

Explanation:
Dictionary me sirf keys check hoti hain, values nahi

Example 5: Login / Validation Use Case

allowed_users = ["admin", "user1", "user2"]

username in allowed_users → True

Explanation:
Username list me present hai, access allow hoga

Use Cases:

  • List
  • String
  • Tuple
  • Set

Important Points:

  • Membership operators sequence ke sath kaam karte hain
  • Result hamesha True ya False hota hai
  • Dictionary me in keys par kaam karta hai, values par nahi

4.7 Identity Operators

Identity Operators ka use memory location (object identity) compare karne ke liye hota hai.

Ye check karte hain ki do variables same object ko refer kar rahe hain ya nahi.

  • is → Same memory location check karta hai
  • is not → Different memory location check karta hai

Basic Example:

a = [1, 2, 3]
b = a
c = [1, 2, 3]

a is b → True

Explanation:
b = a ka matlab hai b aur a same list object ko point kar rahe hain
Dono ka memory address same hai
Isliye result → True

a is c → False

Explanation:
a aur c ki values same dikhti hain
Lekin c ek naya object hai, memory location alag hai
Isliye is ka result → False

a == c → True

Explanation:
== operator sirf values compare karta hai
a aur c dono me same values hain [1, 2, 3]
Isliye result → True

Important Difference:

  • == → Value compare karta hai
  • is → Memory location (object identity) compare karta hai

Example 1: Integer Case

x = 10
y = 10

x is y → True

Explanation:
Python small integers ko memory me reuse karta hai
Isliye x aur y same object ko refer kar sakte hain

Example 2: Different Objects

m = [5, 6]
n = [5, 6]

m == n → True
m is n → False

Explanation:
Values same hain, isliye == True
Memory alag hai, isliye is False

Example 3: is not Operator

a = "hello"
b = "hello"

a is not b → False

Explanation:
Dono same object ko refer kar rahe hain
Isliye is not ka result False

Real-Life Logic:

== ka use tab karo jab values compare karni ho
is ka use tab karo jab exact same object check karna ho

Explanation:

Condition True hone ki wajah se print statement execute hogi
Output: Eligible to vote

Start = 4 (o)

Characters index 4, 3, 2 liye gaye

Important Rule:

Important Points:

9.5 Recursive Functions

Recursive function wo function hota hai jo khud ko hi call karta hai.

Recursion ka use tab hota hai jab problem ko chhoti sub-problems me tod sakte hain.

Two Important Parts:

  • Base Condition
  • Recursive Call

Base condition recursion ko stop karti hai,
aur recursive call function ko dobara call karta hai.

Example: Factorial

def factorial(n):
    if n == 1:
        return 1
    return n * factorial(n - 1)

How This Works (Step-by-Step):

factorial(5) = 5 * factorial(4)
factorial(4) = 4 * factorial(3)
factorial(3) = 3 * factorial(2)
factorial(2) = 2 * factorial(1)
factorial(1) = 1  ← base condition

Ab values wapas return hoti hain:

factorial(2) = 2 * 1 = 2
factorial(3) = 3 * 2 = 6
factorial(4) = 4 * 6 = 24
factorial(5) = 5 * 24 = 120

Final Output:
factorial(5) → 120

Why Base Condition is Important?

Base condition recursion ko stop karti hai.

Agar base condition nahi hogi, to function khud ko infinite times call karega aur program crash ho jayega.

Wrong Example (No Base Condition):

def test():
    return test()

Result:
Infinite recursion → RecursionError

Another Example: Sum of Numbers

def sum_n(n):
    if n == 0:
        return 0
    return n + sum_n(n - 1)

sum_n(5) → 15

Important Points:

  • Har recursive function me base condition zaroori hai
  • Recursive call problem ko chhota karta hai
  • Recursion ko loop se bhi replace kiya ja sakta hai
  • Zyada deep recursion performance slow kar sakta hai

Example: Write Mode

Explanation:

11.2 File Modes (Very Important)

File mode decide karta hai ki file ke saath hum kya operation kar sakte hain (read, write, append, etc.).

Galat mode use karne se data loss ya error ho sakta hai, isliye ye topic bahut important hai.

1) Read Mode (r)

File ko read karne ke liye use hota hai.

  • File ka exist hona mandatory hai
  • Write allowed nahi hota

Example:

f = open("data.txt", "r")

Agar file exist nahi karti → FileNotFoundError

2) Write Mode (w)

File me write karne ke liye use hota hai.

  • File exist nahi karti → new file create hoti hai
  • File exist karti hai → poora data overwrite ho jata hai

Example:

f = open("data.txt", "w")

⚠️ Warning: purana data delete ho jata hai

3) Append Mode (a)

File ke end me data add karne ke liye use hota hai.

  • Purana data safe rehta hai
  • File exist nahi karti → new file create hoti hai

Example:

f = open("data.txt", "a")

4) Read + Write Mode (r+)

File ko read aur write dono kar sakte hain.

  • File ka exist hona zaroori hai
  • Data overwrite ho sakta hai (cursor position par)

Example:

f = open("data.txt", "r+")

5) Write + Read Mode (w+)

File ko pehle overwrite karta hai, phir read aur write dono allow karta hai.

  • Purana data delete ho jata hai
  • File exist nahi ho to create hoti hai

Example:

f = open("data.txt", "w+")

Mode Comparison Summary

  • r → Read only, file must exist
  • w → Write only, overwrite data
  • a → Append only, data safe
  • r+ → Read + Write, no overwrite by default
  • w+ → Write + Read, overwrite data

Common Beginner Mistakes:

  • Read mode me write karna try karna
  • w mode use karke data delete kar dena
  • File exist na hone par r mode use karna

Best Practices:

  • Sirf read ke liye → r
  • Safe data add ke liye → a
  • Overwrite karna ho tabhi → w
  • Always use → with open()

Interview Tip:

w vs a ka difference interview me almost confirm hota hai

13.5 Inheritance

Inheritance ka matlab hota hai ek class ka doosri class ke features (methods, variables) ko reuse karna.

Jis class se inherit kiya jata hai use Parent / Base class kehte hain, aur jo class inherit karti hai use Child / Derived class.

Why Inheritance is Used?

  • Code reuse
  • Duplicate code avoid karna
  • Program ko structured banana

Basic Example:

class Animal:
  def sound(self):
    print("Animal sound")

class Dog(Animal):
  def bark(self):
    print("Dog barks")

d = Dog()
d.sound()
d.bark()

Explanation:
Dog class ne Animal class ko inherit kiya
Isliye Dog object sound() aur bark() dono use kar sakta hai

Types of Inheritance

  • Single Inheritance
  • Multilevel Inheritance
  • Multiple Inheritance

1) Single Inheritance

Jab ek child class sirf ek parent class se inherit karti hai, use Single Inheritance kehte hain.

Example:

class Vehicle:
  def start(self):
    print("Vehicle started")

class Car(Vehicle):
  def drive(self):
    print("Car is driving")

c = Car()
c.start()
c.drive()

Explanation:
Car ne Vehicle ko inherit kiya
start() parent se mila
drive() Car ka khud ka method hai

2) Multilevel Inheritance

Jab ek class doosri class se inherit kare, aur wo class kisi aur class se inherit ho, use Multilevel Inheritance kehte hain.

Example:

class LivingBeing:
  def breathe(self):
    print("Breathing")

class Animal(LivingBeing):
  def eat(self):
    print("Eating")

class Dog(Animal):
  def bark(self):
    print("Barking")

d = Dog()
d.breathe()
d.eat()
d.bark()

Explanation:
Dog → Animal → LivingBeing
Dog object teeno class ke methods access kar sakta hai

3) Multiple Inheritance

Jab ek child class ek se zyada parent classes se inherit karti hai, use Multiple Inheritance kehte hain.

Example:

class Father:
  def skills(self):
    print("Father skills")

class Mother:
  def skills(self):
    print("Mother skills")

class Child(Father, Mother):
  pass

c = Child()
c.skills()

Explanation:
Child ne Father aur Mother dono ko inherit kiya
Python MRO (Method Resolution Order) follow karta hai
Jo class pehle likhi hoti hai, uska method call hota hai

Important Points:

  • Parent class ke methods child class me automatically available hote hain
  • Child class parent ke methods ko override kar sakti hai
  • Multiple inheritance me MRO follow hota hai

Exam Line (Very Important):

  • Inheritance = code reuse
  • IS-A relationship represent karta hai

Interview Tip:

Inheritance improves reusability, par unnecessary inheritance avoid karna chahiye

    return value

  print(i)

14.3 Decorators

Decorator ek aisa function hota hai jo kisi dusre function ke behaviour ko modify karta hai, bina original function ke code ko change kiye.

After function

  • my_decorator ek wrapper function return karta hai

17.3 Concurrency vs Parallelism

Concurrency:

  • Multiple tasks ka progress ek saath
  • Single CPU core par possible
  • Context switching

Parallelism:

  • Tasks actually ek saath run hote hain
  • Multiple CPU cores required

Real-life analogy:

  • Concurrency = ek aadmi 2 kaam switch karke
  • Parallelism = 2 aadmi 2 kaam ek saath

Threading vs Multiprocessing (Interview Table)

Feature Threading Multiprocessing
Unit Thread Process
Memory Shared Separate
GIL Impact Yes No
Best For I/O-bound CPU-bound
Speed Limited True parallel
Memory Use Low High

Common Mistakes (Avoid These)

  • CPU-bound task me threading use karna ❌
  • Shared data without lock ❌
  • Multiprocessing ko lightweight samajhna ❌

One-Line Revision (Exam Ready)

  • Threading = concurrency + shared memory
  • Multiprocessing = parallelism + separate memory
  • GIL = threading limit
  • I/O-bound → threading
  • CPU-bound → multiprocessing
