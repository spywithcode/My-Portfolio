PHASE 0: Programming & Computer Fundamentals

0.1 What is Programming

Programming ka matlab hota hai computer ko step-by-step instructions dena taaki wo koi specific task perform kar sake.

Computer khud se kuch nahi karta, jab tak hum usse clearly na batayein ki kya karna hai.

  • Computer ek machine hai
  • Machine instructions par kaam karti hai
  • Instructions programming language me likhi jaati hain

Programming ka main goal hota hai:

  • Problem solve karna
  • Logic develop karna
  • Automation create karna

0.2 Why Programming is Needed

Programming ke bina computer sirf ek dead machine hoti hai.

  • Software banane ke liye
  • Mobile apps ke liye
  • Websites ke liye
  • Games ke liye
  • AI, Data, Automation ke liye

Real-life example:

Jaise hum kisi insaan ko kaam samjhate hain, waise hi computer ko bhi instructions deni padti hain.

0.3 Human Language vs Machine Language

Human Language

Human language wo hoti hai jo humans aapas me baat karne ke liye use karte hain.

  • Examples: Hindi, English
  • Hum easily samajh sakte hain
  • Computer directly nahi samajhta

Machine Language

Machine language sirf 0 aur 1 (binary) me hoti hai.

  • Computer directly samajhta hai
  • Humans ke liye bahut difficult
  • Error-prone hoti hai

Programming Language

Programming language human aur machine ke beech bridge ka kaam karti hai.

  • Human-friendly syntax
  • Machine understandable form me convert hoti hai
  • Examples: C, Python, Java

0.4 What is an Algorithm

Algorithm kisi bhi problem ko solve karne ke liye step-by-step logical solution hota hai.

  • Finite steps
  • Clear logic
  • Order me likhe gaye steps

Algorithm example (Even or Odd):

Start
Input number
If number % 2 == 0
Print Even
Else
Print Odd
End

0.5 Characteristics of a Good Algorithm

  • Correctness – sahi output de
  • Clarity – steps clear ho
  • Efficiency – kam time aur memory use kare
  • Finiteness – kabhi infinite na ho

0.6 What is a Flowchart

Flowchart algorithm ka graphical (diagram) representation hota hai.

  • Logic easily samajh aata hai
  • Debugging easy hoti hai
  • Planning me help karta hai

Common Flowchart Symbols:

  • Oval – Start / End
  • Rectangle – Process
  • Diamond – Decision
  • Arrow – Flow direction

0.7 What is Pseudocode

Pseudocode ek simple English-like language hoti hai jo program ka logic explain karti hai.

  • Programming language specific nahi hoti
  • Easy to write
  • Easy to understand

Pseudocode example:

Start
Input two numbers
Add numbers
Display result
End

  • Line-by-line execution

Program instructions ka collection hota hai jo computer ko batata hai ki kya karna hai.

  • Input

PHASE 2: Python Execution & Basics

2.1 How Python Executes Code

Python ka execution process sirf simple line-by-line nahi hota. Internally Python ek proper execution flow follow karta hai.

Python Execution Flow:

Source Code (.py)
↓
Bytecode (.pyc)
↓
Python Virtual Machine (PVM)
↓
Output

2.2 Source Code (.py)

Source code wo hota hai jo programmer likhta hai.

  • Human readable hota hai
  • .py extension hoti hai
  • Direct hardware par execute nahi hota

Example:

print("Hello Python")

2.3 Bytecode (.pyc)

Jab Python program run hota hai, pehle Python interpreter source code ko bytecode me convert karta hai.

  • Low-level instructions hoti hain
  • Platform independent hoti hain
  • Human ke liye readable nahi hoti

Bytecode file:

  • .pyc extension
  • __pycache__ folder me store hoti hai

Purpose:

  • Performance improve karna
  • Repeated execution fast banana

2.4 Python Virtual Machine (PVM)

PVM Python ka runtime engine hota hai.

  • Bytecode ko read karta hai
  • Instructions execute karta hai
  • Memory manage karta hai
  • Errors handle karta hai

Simple understanding:

Java ke liye JVM hota hai, Python ke liye PVM hota hai.

2.5 Role of Python Interpreter

Python interpreter ek complete system hota hai jo multiple kaam karta hai.

  • Source code read karta hai
  • Syntax check karta hai
  • Bytecode generate karta hai
  • PVM ke through execution karata hai

Important clarification:

Python interpreted language hai, lekin internally bytecode + virtual machine ka use karta hai.

2.6 First Python Program

print() Function

print() Python ka output function hai.

  • Screen par output show karta hai
  • Multiple values print kar sakta hai

Example:

print("Hello Python")

Output:

Hello Python

Comments wo lines hoti hain jo Python execute nahi karta.

  • Code explanation ke liye

Indentation Python me mandatory hoti hai.

  • Code block define karti hai
  • { } ka use nahi hota
  • 4 spaces recommended hote hain

Example:

Indentation galat hui to error aata hai.

2.9 Errors in Python

Error ka matlab hai program ka galat behave karna ya crash ho jana.

Syntax Error

  • Grammar mistake
  • Program run hi nahi hota

Example:

print("Hello"

Runtime Error

  • Program run hote time error
  • Execution stop ho jata hai

Example:

Logical Error

  • Program run hota hai
  • Error nahi aata
  • Output galat hota hai

Example:

b = 5
print(a - b)

PHASE 3: Variables, Data Types & Memory

3.1 Variables

Variable ek name / reference hota hai jo memory me kisi value ko point karta hai.

Important understanding:

Python me variable container nahi hota, balki value ka reference hota hai.

Example:

a = 10

Yahan:

  • a → variable name
  • 10 → value (object)

3.2 Variable Naming Rules

  • Letters (a–z, A–Z), digits (0–9) aur underscore (_) allowed
  • Digit se start nahi ho sakta
  • Keywords allowed nahi
  • Case-sensitive hote hain

Valid names:

age, total_marks, _count

Invalid names:

1age, total marks, if

3.3 Multiple Assignment

Python me ek hi line me multiple variables assign kar sakte hain.

Example:

a, b, c = 10, 20, 30

Same value assignment:

x = y = z = 5

3.4 Dynamic Typing

Python ek dynamically typed language hai.

  • Variable ka data type pehle define nahi karna padta
  • Runtime par type decide hota hai

Example:

x = 10
x = "Hello"

Exam line:

Python me variable ka data type runtime par change ho sakta hai.

3.5 Data Types (Core)

1) int (Integer)

  • Whole numbers
  • Positive, negative, zero

Example: a = 100

2) float

  • Decimal numbers

Example: pi = 3.14

3) complex

  • Real + imaginary part

Example: z = 3 + 4j

4) bool

  • True / False

Example: is_active = True

Note:

True → 1, False → 0 (internally)

5) str (String)

  • Characters ka sequence
  • Immutable hoti hai

Example: name = "Python"

3.6 Type Checking

Variable ka data type check karne ke liye type() function use hota hai.

Example:

type(a)

3.7 Type Casting

Ek data type ko doosre data type me convert karna type casting kehlata hai.

Explicit Type Casting

Programmer manually karta hai.

  • int()
  • float()
  • str()
  • bool()

Examples:

int("10") → 10
float(5) → 5.0
bool(0) → False

Implicit Type Casting

Python automatically karta hai.

Example:

a = 10
b = 2.5
c = a + b

Result:

c = 12.5

Python me variable value ko copy nahi karta, reference assign karta hai.

PHASE 4: Operators in Python

What are Operators?

Operators wo symbols hote hain jo values par operations perform karte hain.

Simple words:

Operator = kaam karne wala symbol

4.1 Arithmetic Operators

Mathematical calculations ke liye use hote hain.

  • + → Addition
  • - → Subtraction
  • * → Multiplication
  • / → Division (float result)
  • // → Floor Division
  • % → Modulus (remainder)
  • ** → Exponent (power)

Example:

a = 10
b = 3

a + b → 13
a - b → 7
a * b → 30
a / b → 3.33
a // b → 3
a % b → 1
a ** b → 1000

Important logic:

  • / hamesha float deta hai
  • // decimal part hata deta hai

4.2 Relational (Comparison) Operators

Do values ko compare karte hain.

Result hamesha True ya False hota hai.

  • == → Equal to
  • != → Not equal to
  • > → Greater than
  • < → Less than
  • >= → Greater than equal to
  • <= → Less than equal to

Example:

a = 10
b = 20

a == b → False
a < b → True

Common mistake:

  • = → Assignment
  • == → Comparison

4.3 Logical Operators

Multiple conditions ko combine karne ke liye use hote hain.

  • and → Dono True honi chahiye
  • or → Koi ek True ho
  • not → Result reverse karta hai

Example:

a = 10
b = 5

a > 5 and b < 10 → True
a > 15 or b < 10 → True
not(a > 5) → False

4.4 Assignment Operators

Value assign aur update karne ke liye use hote hain.

  • = → Assign
  • += → Add & assign
  • -= → Subtract & assign
  • *= → Multiply & assign
  • /= → Divide & assign
  • //= → Floor divide & assign
  • %= → Modulus & assign
  • **= → Power & assign

Example:

a = 10
a += 5 → 15
a *= 2 → 30

Logic:

a += b ≡ a = a + b

4.5 Bitwise Operators

Binary (0/1) level par kaam karte hain.

  • & → Bitwise AND
  • | → Bitwise OR
  • ^ → Bitwise XOR
  • ~ → Bitwise NOT
  • << → Left Shift
  • >> → Right Shift

Example:

a = 5 (0101)
b = 3 (0011)

a & b → 1
a | b → 7
a ^ b → 6
~a → -6
a << 1 → 10
a >> 1 → 2

Important:

  • Bitwise ≠ Logical
  • Binary me kaam karta hai

4.6 Membership Operators

Check karte hain ki value kisi sequence me present hai ya nahi.

  • in
  • not in

Example:

nums = [1, 2, 3, 4]

2 in nums → True
5 not in nums → True

Use cases:

  • List
  • String
  • Tuple
  • Set
  • Dictionary

4.7 Identity Operators

Memory location (object identity) compare karte hain.

  • is
  • is not

Example:

a = [1, 2, 3]
b = a
c = [1, 2, 3]

a is b → True
a is c → False
a == c → True

Difference:

  • == → Value compare
  • is → Memory compare

4.8 Operator Summary Table

  • Arithmetic → Calculations
  • Relational → Comparison
  • Logical → Conditions
  • Assignment → Update
  • Bitwise → Binary operations
  • Membership → in / not in
  • Identity → is / is not

4.9 Common Exam & Interview Traps

  • = vs == confusion
  • and/or vs &/| confusion
  • / vs // confusion
  • is vs == confusion

4.10 One-Line Revision

  • Arithmetic → Math
  • Relational → Compare
  • Logical → Combine conditions
  • Assignment → Update values
  • Bitwise → Binary
  • Membership → in / not in
  • Identity → Memory compare

PHASE 5: Control Flow (Decision Making)

What is Control Flow?

Control Flow ka matlab hai program kis condition par kaunsa code execute karega.

Simple words:

Control Flow = decision lene ka tareeqa

Python me decision making:

  • Conditions
  • Indentation

5.1 if Statement

Jab condition True hoti hai tab code execute hota hai.

Syntax:

if condition:
    code

Example:

age = 18
if age >= 18:
    print("Eligible to vote")

Important points:

  • Colon (:) mandatory
  • Indentation mandatory
  • Sirf ek condition check hoti hai

5.2 if-else Statement

Jab condition True ho to ek block, False ho to doosra block execute hota hai.

Syntax:

if condition:
    code_if_true
else:
    code_if_false

Example:

num = 5
if num % 2 == 0:
    print("Even")
else:
    print("Odd")

Logic:

  • if → True case
  • else → False case

5.3 if-elif-else Ladder

Jab multiple conditions ko ek ke baad ek check karna ho, tab if-elif-else use hota hai.

Syntax:

if condition1:
    code
elif condition2:
    code
else:
    default_code

Example:

marks = 72
if marks >= 90:
    print("Grade A")
elif marks >= 75:
    print("Grade B")
elif marks >= 50:
    print("Grade C")
else:
    print("Fail")

Rules:

  • Conditions top to bottom check hoti hain
  • Sirf pehli True condition execute hoti hai
  • else optional hota hai

5.4 Nested if

Jab ek if ke andar doosra if hota hai, use nested if kehte hain.

Syntax:

if condition1:
    if condition2:
        code

Example:

age = 20
has_id = True
if age >= 18:
    if has_id:
        print("Entry allowed")

Logic:

  • Pehle outer condition check hoti hai
  • Phir inner condition

5.5 Real-Life Decision Examples

Login System

username = "admin"
password = "1234"
if username == "admin" and password == "1234":
    print("Login successful")
else:
    print("Invalid credentials")

Discount System

amount = 4500
if amount >= 5000:
    print("20% discount")
elif amount >= 3000:
    print("10% discount")
else:
    print("No discount")

Traffic Signal

signal = "red"
if signal == "red":
    print("Stop")
elif signal == "yellow":
    print("Wait")
elif signal == "green":
    print("Go")
else:
    print("Invalid signal")

5.6 Common Mistakes (Avoid These)

  • Indentation galat ❌
  • Colon (:) bhoolna ❌
  • = aur == ka confusion ❌
  • elif ko else if likhna ❌

5.7 Quick Comparison (Exam Point)

  • if → Single condition
  • if-else → Two-way decision
  • if-elif-else → Multiple conditions
  • Nested if → Condition ke andar condition

5.8 One-Line Revision (Exam Ready)

  • Control flow = decision making
  • if = condition true
  • else = condition false
  • elif = multiple conditions
  • Nested if = multi-level decision
  • Indentation = Python ka backbone

PHASE 6: Loops (Iteration)

What is a Loop?

Loop ka kaam hota hai same code ko baar-baar repeat karna jab tak condition satisfy hoti rahe.

Simple words:

Loop = repetition with logic

Python me loops mainly do type ke hote hain:

  • for loop
  • while loop

6.1 for Loop

for loop tab use hota hai jab hume pehle se pata ho loop kitni baar chalega.

Syntax:

for variable in sequence:
    code

Example:

for i in range(5):
    print(i)

Output:

0 1 2 3 4

Sequence kya ho sakta hai:

  • range()
  • list
  • string
  • tuple
  • set

Example (string loop):

for ch in "Python":
    print(ch)

Key points:

  • Sequence-based loop
  • Counter automatically update hota hai

6.2 range() Function

range() numbers ka sequence generate karta hai. Ye mostly for loop ke saath use hota hai.

Forms:

  • range(stop)
  • range(start, stop)
  • range(start, stop, step)

Examples:

range(5) → 0 to 4
range(1, 6) → 1 to 5
range(1, 10, 2) → 1, 3, 5, 7, 9

Example:

for i in range(2, 11, 2):
    print(i)

6.3 while Loop

while loop tab use hota hai jab exact number of iterations pata na ho.

Syntax:

while condition:
    code

Example:

i = 1
while i <= 5:
    print(i)
    i += 1

Important points:

  • Condition True rahegi to loop chalta rahega
  • Condition False hote hi loop stop

Warning:

Condition update na hui to infinite loop ban jata hai.

6.4 Nested Loops

Jab ek loop ke andar doosra loop hota hai, use nested loop kehte hain.

Syntax:

for i in range(x):
    for j in range(y):
        code

Example:

for i in range(1, 4):
    for j in range(1, 4):
        print(i, j)

Use cases:

  • Pattern printing
  • Tables
  • Matrix operations

Logic:

  • Outer loop → rows
  • Inner loop → columns

6.5 Loop Control Statements

break

Loop ko turant stop kar deta hai.

Example:

for i in range(10):
    if i == 5:
        break
    print(i)

continue

Current iteration skip karke next iteration par chala jata hai.

Example:

for i in range(5):
    if i == 2:
        continue
    print(i)

pass

pass ek empty statement hai. Ye syntax complete karta hai.

Example:

for i in range(5):
    pass

else with Loop (Python Special)

Loop normally complete ho jaye to else execute hota hai.

Agar break laga ho to else execute nahi hota.

Example:

for i in range(5):
    if i == 10:
        break
else:
    print("Loop completed successfully")

6.6 Common Mistakes (Avoid These)

  • while loop me condition update bhoolna ❌
  • range ke stop ko include samajhna ❌
  • break aur continue confuse karna ❌
  • Indentation galat ❌

6.7 for vs while (Exam Point)

  • for → fixed iterations
  • while → condition-based
  • for → cleaner syntax
  • while → flexible but risky

6.8 One-Line Revision (Exam Ready)

  • Loop = repetition
  • for = sequence-based
  • while = condition-based
  • range() = number generator
  • Nested loop = loop ke andar loop
  • break = stop loop
  • continue = skip iteration
  • pass = empty block

PHASE 7: Strings (Very Important)

What is a String?

String characters ka sequence hoti hai.

Real-life use:

  • Name
  • Password
  • Messages
  • File content
  • User input

Python me string immutable hoti hai.

7.1 String Creation

Python me string quotes ke andar likhi jaati hai.

Ways to create string:

Single quotes:

name = 'Python'

Double quotes:

msg = "Hello World"

Triple quotes (multi-line string):

text = """
This is
a multi-line
string
"""

Important:

  • Single aur double quotes same behave karte hain
  • Triple quotes multi-line text ke liye

7.2 String Indexing

Har character ka ek index hota hai.

Index hamesha 0 se start hota hai.

Example:

s = "Python"
Index: 0 1 2 3 4 5

Positive indexing:

  • s[0] → P
  • s[3] → h

Negative indexing:

  • s[-1] → n
  • s[-2] → o

7.3 String Slicing

Slicing ka matlab string ka ek part nikalna.

Syntax:

string[start : end : step]

Examples:

s = "Python"
s[0:4] → Pyth
s[2:] → thon
s[:3] → Pyt
s[::2] → Pto
s[::-1] → nohtyP

Important rule:

  • End index include nahi hota

7.4 String Immutability

Python me string immutable hoti hai.

Matlab:

  • String ka character directly change nahi kar sakte

Invalid example:

s = "Python"
s[0] = 'J'

Correct way:

s = "Jython"

Reason:

Modification par new string object create hota hai.

7.5 Common String Methods

lower() / upper()

Case change karte hain.

s = "Python"
s.lower() → python
s.upper() → PYTHON

split()

String ko list me tod deta hai.

s = "I love Python"
s.split() → ['I', 'love', 'Python']

join()

List ke elements ko string me jodta hai.

words = ['I', 'love', 'Python']
" ".join(words) → I love Python

Note:

  • join() string ka method hota hai

replace()

String ka part replace karta hai.

s = "I love Java"
s.replace("Java", "Python")

find() / index()

Substring ka index batate hain.

s = "Python"
s.find('t') → 2
s.find('x') → -1

s.index('t') → 2
s.index('x') → Error

Difference:

  • find() → -1 return karta hai
  • index() → error deta hai

count()

Character ya substring kitni baar aaya hai.

s = "banana"
s.count('a') → 3

strip()

Extra spaces remove karta hai.

s = " hello "
s.strip() → hello

Types:

  • strip() → both sides

14.6 map(), filter(), reduce()

  • filter → select
  • reduce → combine

  • Closure concept skip karna ❌
  • reduce import bhoolna ❌

One-Line Revision (Exam Ready)

  • Iterator = next value provider
  • Generator = yield based function
  • Decorator = function modifier
  • Closure = function + memory
  • Comprehension = short loops
  • map = transform
  • filter = select
  • reduce = single result

16.2 Reference Counting

Reference counting Python ka primary garbage collection technique hai.

Kaise kaam karta hai:

  • New reference → count +1
  • Reference delete → count -1
  • Count = 0 → object delete

Example:

a = []
b = a # count = 2
del b # count = 1
del a # count = 0 → memory free

Circular Reference Problem

Jab objects ek dusre ko reference karte hain.

Example:

a = {}
b = {}
a["b"] = b
b["a"] = a

Problem:

  • Reference count zero nahi hota

Solution:

  • Python ka Cyclic Garbage Collector

16.4 Shallow Copy vs Deep Copy

Shallow Copy

  • Outer object copy hota hai
  • Inner objects shared rehte hain

Example:

import copy

a = [[1,2],[3,4]]
b = copy.copy(a)
b[0][0] = 99

Result:

  • a bhi change ho jayega

Deep Copy

  • Outer + inner sab naya copy
  • No shared reference

Example:

c = copy.deepcopy(a)
c[0][0] = 50

Result:

  • a change nahi hoga

Exam Line:

  • Shallow = shared reference
  • Deep = independent objects

Why Concurrency & Parallelism?

  • Performance improve karna
  • I/O aur CPU tasks ko sahi tool se handle karna

Key Concepts (Foundation)

  • Lightweight unit of execution
  • Same process ke andar run hota hai

  • Independent execution unit
  • Apni memory hoti hai

17.1 Multithreading

Multithreading = ek hi process ke andar multiple threads.

Best for:

  • I/O-bound tasks
  • File read/write
  • Network requests

Threading module:

import threading

Basic Thread Example

import threading
import time

def task():
  print("Task start")
  time.sleep(2)
  print("Task end")

t1 = threading.Thread(target=task)
t2 = threading.Thread(target=task)

t1.start()
t2.start()

t1.join()
t2.join()

print("Done")

Important Points:

  • Threads same memory share karte hain
  • Context switching fast hota hai
  • Race condition ka risk hota hai

Thread Synchronization (Locks)

Shared data ko protect karne ke liye lock use hota hai.

Example:

lock = threading.Lock()

  with lock:
    print("Critical section")

Exam Line:

  • Lock race condition se bachata hai

GIL Recap (Very Important)

  • GIL = ek time par sirf ek thread bytecode execute karta hai
  • CPU-bound tasks me threading slow hoti hai
  • I/O-bound tasks me threading effective hoti hai

17.2 Multiprocessing

Multiprocessing = multiple processes ek saath.

Best for:

  • CPU-bound tasks
  • Heavy computations

multiprocessing module:

import multiprocessing

Basic Multiprocessing Example

import multiprocessing
import time

def task():
  print("Process running")
  time.sleep(2)

p1 = multiprocessing.Process(target=task)
p2 = multiprocessing.Process(target=task)

p1.start()
p2.start()

p1.join()
p2.join()

print("Done")

Important Points:

  • Har process ka apna Python interpreter hota hai
  • GIL ka koi issue nahi
  • Memory usage zyada hoti hai

Inter-Process Communication (IPC)

Processes memory share nahi karte, isliye IPC chahiye.

Common IPC tools:

  • Queue
  • Pipe
  • Shared Value / Array

Queue Example:

from multiprocessing import Process, Queue

def worker(q):
  q.put("Data from process")

p = Process(target=worker, args=(q,))
p.start()
print(q.get())
p.join()

17.3 Concurrency vs Parallelism

Concurrency:

  • Multiple tasks ka progress ek saath
  • Single CPU core par possible
  • Context switching

Parallelism:

  • Tasks actually ek saath run hote hain
  • Multiple CPU cores required

Real-life analogy:

  • Concurrency = ek aadmi 2 kaam switch karke
  • Parallelism = 2 aadmi 2 kaam ek saath

Best For I/O-bound CPU-bound

Memory Use Low High

  • CPU-bound task me threading use karna ❌
  • Shared data without lock ❌
  • Multiprocessing ko lightweight samajhna ❌

One-Line Revision (Exam Ready)

  • Threading = concurrency + shared memory
  • Multiprocessing = parallelism + separate memory
  • GIL = threading limit
  • I/O-bound → threading
  • CPU-bound → multiprocessing
